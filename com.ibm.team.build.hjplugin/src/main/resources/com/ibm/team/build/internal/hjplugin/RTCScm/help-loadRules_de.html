Wählen Sie eine Datei mit Laderegeln aus, um anzugeben, wie der Repository-Arbeitsbereich geladen werden soll. Wenn dynamische Laderegeln mithilfe einer Erweiterung angegeben wurden, haben diese Vorrang vor den hier angegebenen Laderegeln.

Um Laderegeln anzuwenden, listen Sie sie im folgenden JSON-Format auf, speichern Sie die Liste in einer Datei auf einem Datenträger und geben Sie den Pfad zur Datei in diesem Feld an. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

Sie können auch "componentId" (Komponenten-ID) anstelle von "componentName" (Komponentenname) angeben. Bei "componentId" handelt es sich um die Element-ID (UUID) der Komponente. Wenn Sie sowohl "componentName" als auch "componentId" angeben, erfolgt die Suche nur für "componentId."

Der Wert für "filePath" (Dateipfad) ist der vollständige Pfad zur Datei mit Laderegeln im Repository, d. h. der ferne Pfad. Sie können auch "fileItemId" (Dateielement-ID) anstelle von "filePath" (DateiPfad) angeben. Bei "fileItemId" handelt es sich um die Element-ID (UUID) der Datei mit Laderegeln. Wenn Sie den Dateipfad angeben, verwenden Sie normale Schrägstriche (/) zum Trennen der Pfadsegmente. Wenn Sie sowohl "filePath" als auch "fileItemId" angeben, erfolgt die Suche nur für "fileItemId."
