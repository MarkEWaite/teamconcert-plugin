Выберите файл правил загрузки, определяющий способ загрузки рабочей области хранилища. Если с помощью расширения были заданы динамические правила загрузки, то они имеют больший приоритет, чем указанные здесь правила.

Для применения правил загрузки создайте их список в следующем формате JSON, сохраните список в файле на диске и укажите путь к файлу в данном поле. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

Вместо "componentName" можно указать "componentId". "componentId" задает присвоенный компоненту ИД элемента (UUID). Если задано как "componentName", так и "componentId", используется только "componentId".

Значение "filePath" задает полный путь к файлу правил загрузки в хранилище, то есть удаленный путь. Вместо "filePath" можно указать "fileItemId". "fileItemId" задает ИД элемента (UUID), присвоенный файлу правил загрузки. Сегменты пути к файлу следует разделять косой чертой (/). Если задано как "filePath", так и "fileItemId", будет использоваться только "fileItemId".
