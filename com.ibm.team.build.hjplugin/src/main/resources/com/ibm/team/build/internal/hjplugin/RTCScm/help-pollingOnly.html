When this option set to true, the plugin will perform polling only but will not accept or load the repository workspace.

Note:If you select this option, then you must ensure that changes are accepted to the workspace as part of the build, within the next polling interval.

If this option is is set to true, then RTCScm will poll the repository workspace only and not perform the following actions,

  • accept
  • load
  • recording change log in the build
  • updating Jenkins build with plugin specific data

Evolution and backward compatibility

  • For jobs created before 2.3.0, the option's value will be false.
  • For new jobs, the option's value will be false.

Constraints

  • Supported for pipeline jobs.
  • Supported for build definition and build workspace types.
