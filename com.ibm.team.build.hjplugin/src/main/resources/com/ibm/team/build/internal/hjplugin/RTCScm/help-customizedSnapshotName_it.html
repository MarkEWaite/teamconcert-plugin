Specifica il nome fornito all'istantanea creata dopo l'accettazione delle modifiche.

Questo campo può essere impostato su qualsiasi stringa specificata dall'utente, con riferimenti alle variabili di ambiente ed ai parametri di build. Si consiglia di inserire almeno una variabile il cui valore sia univoco per ciascuna build, ad esempio BUILD_NUMBER, BUILD_ID o BUILD_URL, per garantire che build consecutive creino istantanee con nomi differenti. Se questo campo è vuoto, viene utilizzato un nome istantanea predefinito. Per i lavori che utilizzano una definizione build, il nome dell'istantanea è "${buildDefinitionId}_#${BUILD_NUMBER}" per impostazione predefinita. Per i lavori che utilizzano uno stream o uno spazio di lavoro del repository, il nome dell'istantanea è "${JOB_NAME}_#${BUILD_NUMBER}" per impostazione predefinita.

Esempio: per un lavoro denominato "JUnit Integration Build", se questo campo è impostato su "${JOB_NAME}_#${BUILD_NUMBER}_Snapshot", il nome dell'istantanea creata dalla prima build sarà "JUnit Integration Build_#1_Snapshot".
