Sélectionnez un fichier de règles de chargement pour indiquer comment charger l'espace de travail de référentiel. Si des règles de chargement dynamiques ont été indiquées en utilisant une extension, elles ont la priorité sur celles indiquées ici.

Pour appliquer les règles de chargement, listez-les au format JSON suivant, sauvegardez la liste dans un fichier sur le disque et indiquez le chemin d'accès au fichier dans cette zone. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

Vous pouvez également indiquer "componentId" au lieu de "componentName." "componentId" est l'ID élément (identificateur unique universel) du composant. Si vous indiquez à la fois "componentName" et "componentId", la recherche se produit uniquement par "componentId."

La valeur "filePath" est le chemin d'accès complet au fichier de règles de chargement, à savoir, le chemin d'accès distant. Vous pouvez également indiquer "fileItemId" au lieu de "filePath." "fileItemId" est l'ID élément (identificateur unique universel) du fichier de règles de chargement. Lorsque vous indiquez le chemin d'accès au fichier, utilisez des barres obliques (/) pour en séparer les segments. Si vous indiquez à la fois "filePath" et "fileItemId," la recherche se produit uniquement par "fileItemId."
