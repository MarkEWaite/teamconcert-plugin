Selezionare un file delle regole di caricamento per specificare il modo in cui caricare lo spazio di lavoro del repository. Se sono state specificate le regole di caricamento dinamico utilizzando un'estensione, tali regole hanno precedenza rispetto a quelle specificate in questo punto.

Per applicare le regole di caricamento, elencarle nel seguente formato JSON, salvare l'elenco in un file su disco e specificare il percorso del file in questo campo. 

  {
    "loadRules":[
      {
        "componentName":"JUnit",
        "filePath":"JUnit Project/junit-project.loadrule"
      },
      {
        "componentName":"Releng",
        "filePath":"Releng Project/releng-project.loadrule"
      }
    ]
  }

È anche possibile specificare "componentId" invece di "componentName." "componentId" è l'ID elemento (UUID) del componente. Se "componentName" e "componentId" vengono entrambi specificati, la ricerca viene eseguita solo per "componentId."

Il valore "filePath" è il percorso completo del file delle regole di caricamento nel repository; vale a dire, il percorso remoto. È anche possibile specificare "fileItemId" invece di "filePath." "fileItemId" è l'ID elemento (UUID) del file delle regole di caricamento. Quando viene specificato il percorso file, utilizzare le barre (/) per separare i segmenti del percorso. Se "filePath" e "fileItemId" vengono entrambi specificati, la ricerca viene eseguita solo per "fileItemId."
